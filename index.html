Gitの罠

前回の更新の時にコミットに本名が書かれている事に気がついた。OS XでのGitの初期設定ではなんと本名がコミットに含まれてしまうのだ。これはまずいので、ホームディレクトリの.git configに

[user]
name = 設定したい名前

を追加してコミットに含まれる名前を変更した。しかしGitHub上の本名の含まれたコミットは残ったままだ。これを消そうと色々やったのだが途中で訳が分からなくなった。もうこうなったらリポジトリごと作り直そうと私は思った。

……思ったのだがローカルのリポジトリが消せない。どう消せば良いのか調べたら普通にrm -rfで消せば良いようだ。あとはGitHub上のリポジトリをSettingsから消して、リポジトリを作り直すだけだ。

こうしてリポジトリは作り直され、このサイトも無事復活したのだった。めでたしめでたし。

サイトのコミットスクリプト

このサイトをコミットするためのシェルスクリプトを作った。これでコミットが楽になる。

GitHubへのpush

GitHubにpushする時に毎回ユーザー名とパスワードを求められるのが面倒なので、sshを使おうと鍵を生成したり何なりしたのだが、どうやらhttpsでもパスワードを保存出来たらしい。下のコマンドでパスワードが保存されるようになる。

$ git config --global credential.helper osxkeychain
